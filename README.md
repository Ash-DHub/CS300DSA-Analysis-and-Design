# CS300DSA-Analysis-and-Design
Portfolio showcasing data structures and algorithms projects completed in CS 300 
This repository contains portfolio artifacts from my work in CS 300: DSA Analysis and Design. These projects demonstrate what I learned about data structures, algorithms, and how selecting the right solution can improve the overall performance and organization of a program. 

## Reflection 

### What was the problem you were solving in the projects for this course? 
Throughout this course, I learned how different data structures can affect the performance of a program. In Project One, I compared several data structures by analyzing their run-time performance and memory usage to determine which one would be the best choice for the given requirements. In Project Two, I developed a program that read course information from a file, stored the data, sorted the courses in alphanumeric order, and allowed users to search for individual courses. Both projects helped me better understand how important it is to choose the right data structure before writing code. 

### How did you approach the problem? Consider why data structures are important to understand. 
Before writing any code, I tried to understand the project requirements and think about which data structure would work best for the problem. Throughout this course, I realized that choosing the right data structure can make a significant difference in how efficiently a program runs. Learning about vectors, hash tables, binary search trees, and their advantages and disadvantages helped me better understand why different problems require different solutions. 

### How did you overcome any roadblocks you encountered while going through the activities or project? 
One of the biggest challenges I faced was organizing my program so everything worked together correctly. I also spent time troubleshooting issues with reading files, sorting the course information, and making sure the output matched the assignment requirements. Whenever I got stuck, I went back to the project instructions, reviewed my notes and the course materials, and tested one section of my code at a time until I found the problem. Breaking the project into smaller steps made it much easier to manage. 

### How has your work on this project expanded your approach to designing software and developing programs? 
This course taught me that planning is just as important as writing the code itself. Instead of jumping straight into coding, I now spend more time thinking about how the program should be organized and which data structures and algorithms make the most sense for the problem. I found that taking the time to plan first usually made the implementation process much smoother. 

### How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable? 
Working through these projects helped me realize how important it is to write clean and organized code. I tried to keep my functions focused on one task, use meaningful variable names, and avoid repeating the same logic whenever possible. I also learned that writing code this way makes it much easier to debug, update, and expand later. Moving forward, I plan to continue using these practices because they make my programs easier to understand and maintain.
